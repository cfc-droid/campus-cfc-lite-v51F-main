PEA — Cálculo de estadísticas (guía metodológica completa)

Este documento explica, de forma ultra detallada, cómo se calcula cada una de las 17 estadísticas del sistema PEA.
El objetivo es que el usuario pueda entender exactamente qué significa cada cuadro, cuál es su universo, qué reglas aplica y cómo se obtienen los números.

Resumen rápido (lectura práctica)

  • Cada estadística responde una pregunta concreta (si no responde una pregunta, no existe).
  • Si aparece “datos insuficientes”, el sistema no calcula y no hay conclusión posible.
  • Con muestras chicas, los porcentajes son volátiles (1 caso puede ser 100%).
  • PEA es descriptivo: muestra qué ocurrió, no qué “debería ocurrir”.
  • Co-ocurrencia ≠ causalidad: que A aparezca con B no significa que A cause B.

Qué es PEA y qué hacen (y NO hacen) estas estadísticas

Qué hacen

  • Ordenan registros operativos en una estructura ANTES → DURANTE → DESPUÉS.
  • Calculan frecuencias (cuántas veces aparece algo) y porcentajes (qué proporción representa).
  • Muestran co-ocurrencias: “esto apareció junto con aquello” (por ejemplo: una acción DURANTE co-ocurre con un cierre GANADA o PERDIDA).
  • Permiten auditoría retrospectiva: detectar patrones de repetición y consistencia.
  • Respetan filtros del usuario (fechas, instrumento, activo, dirección, estado, pensamiento, acción, etc.) según aplique a cada estadística.

Qué NO hacen

  • No interpretan resultados ni emociones.
  • No aconsejan qué hacer.
  • No establecen causalidad (correlación ≠ causa).
  • No predicen resultados futuros.
  • No reemplazan criterio operativo, formación, gestión del riesgo, ni evaluación profesional.

Advertencias metodológicas y de uso responsable

Advertencia metodológica (núcleo):
Si una estadística indica “datos insuficientes”, el sistema no fuerza conclusiones ni “infiere” resultados.
El cuadro puede mostrarse igualmente por integridad estructural (para que la interfaz conserve el formato y el usuario vea qué faltaría completar).

Transparencia: este documento es parte del sistema PEA y existe para garantizar transparencia metodológica.

Limitación de responsabilidad (importante):

  • PEA es un sistema de auditoría operativa. No es un sistema de diagnóstico, terapia, asesoramiento médico, psicológico o financiero.
  • Los resultados dependen de la calidad del dato cargado por el usuario (completitud, consistencia, etiquetas).
  • Con muestras pequeñas, los porcentajes pueden ser volátiles (por ejemplo, 1 evento puede representar 100%).
  • El usuario es responsable de cómo interpreta y utiliza la información. PEA no garantiza resultados.
  • Si el sistema se usa en contextos de trading/finanzas, esto no es consejo financiero ni recomendación de inversión. Es auditoría retrospectiva.
  • Si el sistema se usa para estados emocionales, esto no es evaluación clínica. Es una etiqueta operativa autodeclarada.

Definiciones base (vocabulario obligatorio)

  • Registro: una fila cargada por el usuario. Contiene fecha, momento (ANTES/DURANTE/DESPUÉS), y otros campos (pensamiento, acción, estado/emoción, intensidad, dirección, instrumento, etc.).
  • Momento:
    • ANTES: lo previo a la operación (pensamiento/condición previa).
    • DURANTE: lo que ocurre en ejecución (acciones).
    • DESPUÉS: cierre/resultados/estado posterior (incluye resultado operativo).
  • Resultado operativo: etiqueta del cierre (por ejemplo: GANADA, PERDIDA, BE, NA). Por diseño, el resultado “vive” en DESPUÉS.
  • Dirección: COMPRA o VENTA (BUY/SELL). Se usa para distinguir operaciones del mismo día.
  • Pensamiento: texto/etiqueta (o lista) asociada a ANTES.
  • Acción: texto/etiqueta (o lista) asociada a DURANTE.
  • Estado (E) / Emoción: texto/etiqueta (o lista) asociada típicamente a DESPUÉS.
  • Intensidad: valor numérico (típicamente 1 a 5) asociado a DESPUÉS.
  • Estado del registro: auditoría de calidad del registro (VALIDO, CORREGIDO, ANULADO). Por regla general, ANULADO se excluye de métricas.
  • Co-ocurrencia: “X aparece junto con Y” en el universo elegido (no implica causalidad).

Unidades de análisis (esto explica casi todas las dudas)

  • Unidad = DÍA: la estadística cuenta “días” (ej.: cobertura, índices 8/9/10). Un día vale 1, aunque tengas 10 registros.
  • Unidad = REGISTRO: cuenta registros completos (ej.: intensidad en DESPUÉS).
  • Unidad = OCURRENCIA de etiqueta: cuenta cada aparición de pensamiento/acción/estado (si aparece 3 veces, cuenta 3).

Regla “primer / último registro” (auditoría temporal)

  • Primer ANTES del día: cuando una estadística necesita 1 ANTES por fecha, se toma el primer ANTES (orden temporal interno).
  • Primer DURANTE del día: igual para DURANTE.
  • Último DESPUÉS del día (+ dirección): para estadísticas 15/16, se usa el último DESPUÉS del mismo día y misma dirección para definir el cierre.
  • Si tu día tiene varios ANTES/DURANTE/DESPUÉS, el sistema no promedia por defecto: aplica estas reglas para mantener consistencia.

Principios base (reglas no negociables)

  • Conjuntos operativos (SETs):
    • SET_INTERFERENCIA: lista de acciones que el sistema considera “interferentes” (definida en configuración del sistema).
    • SET_CUMPLIMIENTO: lista de acciones que el sistema considera “alineadas al plan / cumplimiento” (definida en configuración del sistema).
    • Si una acción no está en el set correspondiente, no cuenta para esa métrica, aunque “parezca” interferencia/cumplimiento.
    • Estos sets pueden evolucionar por versión: la estadística siempre usa la configuración vigente.
  • Filtro de calidad: el sistema trabaja con registros cuyo estado sea VALIDO o CORREGIDO (CORREGIDO cuenta como VALIDO). ANULADO se excluye.
  • Resultado operativo: para análisis por resultado, la clasificación final se basa en DESPUÉS:
    • En estadísticas “directas de DESPUÉS”, se usa el resultado del mismo registro DESPUÉS.
    • En estadísticas de ANTES o DURANTE, el resultado se hereda desde DESPUÉS usando reglas de cruce (por fecha o por fecha+dirección).
  • Respeto de filtros: en general, el universo usa los filtros activos. Si no hay registros tras filtrar, se muestra “evidencia insuficiente” (no se inventa universo).
  • Unidad de conteo: según la estadística, la unidad puede ser:
    • DÍA (por ejemplo, cobertura).
    • REGISTRO (por ejemplo, intensidad promedio de DESPUÉS).
    • ETIQUETA (pensamiento, acción, estado) por ocurrencias.
  • Ocurrencias vs. “unidades únicas”: cuando una etiqueta se repite, cuenta tantas veces como aparezca. Si un registro contiene “A, B”, cuenta 1 para A y 1 para B.
  • Reglas de cruce entre momentos:
    • Algunas estadísticas cruzan por fecha (día).
    • Otras cruzan por fecha + dirección (COMPRA/VENTA) para evitar mezclar operaciones distintas en el mismo día.

Casos límite y reglas de exclusión (por qué algo NO aparece)

  • Si falta el campo requerido (ej.: falta intensidad en estadísticas emocionales), ese registro no entra al universo de esa estadística.
  • Si no existe DESPUÉS clasificable (GANADA/PERDIDA) para ese cruce, no hay puente posible y ANTES/DURANTE quedan fuera.
  • Si el registro está ANULADO, se excluye siempre (aunque tenga datos “buenos”).
  • Si los filtros dejan el universo vacío, el sistema muestra datos insuficientes y no “rellena”.
  • Si un registro tiene múltiples etiquetas (ej.: “A, B, C”), cuenta 1 ocurrencia por cada etiqueta.
  • Filas “—” en Top: aparecen por Top fijo (estructura), no porque haya datos ocultos.

Nota APB sobre porcentajes y rankings:
Los porcentajes siempre se calculan sobre el universo real de esa estadística. Si el universo tiene 1 sola ocurrencia, esa ocurrencia será 100%. Si una etiqueta no aparece, es porque no entró al universo (filtros, falta de campo, falta de DESPUÉS clasificable, ANULADO, etc.).

Cómo leer porcentajes y rankings en PEA (regla general)

Este bloque aplica a todas las estadísticas del sistema y existe para evitar interpretaciones erróneas. Antes de sacar conclusiones, mirá siempre: universo → filtros → tamaño de muestra.

  • 1) Todo porcentaje es relativo a un universo específico
    Un 80% no significa “80% de todo tu historial”, sino 80% del universo que esa estadística definió (por ejemplo: días con cierre clasificable, registros DESPUÉS válidos, ocurrencias de etiqueta, etc.).
  • 2) 100% con N=1 es correcto, pero no es un patrón estable
    Si el universo tiene 1 caso y algo ocurrió, el sistema mostrará 100%. Eso es matemáticamente correcto, pero significa: “en esta muestra mínima ocurrió siempre”, no “esto pasa siempre”.
  • 3) Ranking (Top) ≠ recomendación
    Un Top solo dice qué apareció más. No significa que sea “bueno”, “malo”, “mejor” o “peor”. PEA es descriptivo, no prescriptivo.
  • 4) Que algo no aparezca no significa que el sistema lo esconda
    Si una etiqueta no aparece en un cuadro, normalmente es por una de estas causas:
    • no hubo ocurrencias reales en el universo,
    • los filtros dejaron fuera esos registros,
    • faltó un campo requerido (ej.: intensidad/dirección),
    • no existió DESPUÉS clasificable (GANADA/PERDIDA) para hacer el cruce,
    • el registro estaba ANULADO.
  • 5) Co-ocurrencia ≠ causalidad
    Que una acción/pensamiento aparezca más en GANADAS o PERDIDAS significa: “apareció junto con ese resultado en este universo”. No significa que lo haya causado.
  • 6) Ocurrencias vs. días: cuidado con la unidad de conteo
    Algunas estadísticas cuentan días (un día vale 1), otras cuentan registros y otras cuentan ocurrencias de etiquetas. Dos cuadros pueden mostrar “%” y aun así no ser comparables si la unidad de conteo es distinta.
  • 7) Regla práctica para usuarios
    Si un resultado te sorprende, revisá en este orden: (a) filtros activos → (b) universo de la estadística → (c) tamaño de muestra (N) → (d) reglas de cruce (fecha vs fecha+dirección) → (e) recién ahí interpretá.

Índice de las 17 estadísticas (nombre + explicación breve)

  1. 1/17 — Cobertura por día (faltantes de los 3 registros): verifica si cada día tiene ANTES + DURANTE + DESPUÉS cargados.
  2. 2/17 — Top Pensamientos en Pérdidas (ANTES): ranking de pensamientos más frecuentes en días que cerraron PERDIDA.
  3. 3/17 — Top Acciones en Pérdidas (DURANTE): ranking de acciones más frecuentes en días que cerraron PERDIDA.
  4. 4/17 — Top Emociones en Pérdidas (DESPUÉS): ranking de estados/emociones más frecuentes en cierres PERDIDA.
  5. 5/17 — Top Pensamientos en Ganadas (ANTES): ranking de pensamientos más frecuentes en días que cerraron GANADA.
  6. 6/17 — Top Acciones en Ganadas (DURANTE): ranking de acciones más frecuentes en días que cerraron GANADA.
  7. 7/17 — Top Emociones en Ganadas (DESPUÉS): ranking de estados/emociones más frecuentes en cierres GANADA.
  8. 8/17 — Índice de Interferencia (GANADA vs PERDIDA): compara % de días con “interferencia” DURANTE según resultado final.
  9. 9/17 — Índice de Cumplimiento (GANADA vs PERDIDA): compara % de días con acciones de “cumplimiento” DURANTE según resultado final.
  10. 10/17 — Balanza Cumplimiento vs Interferencia: vista comparativa de ambos índices lado a lado por resultado.
  11. 11/17 — Top Interferencias por Resultado (DURANTE): desglose descriptivo de interferencias DURANTE separadas por GANADA/PERDIDA.
  12. 12/17 — Intensidad promedio + picos por resultado: compara intensidad emocional (promedio y picos 4–5) en DESPUÉS por GANADA/PERDIDA.
  13. 13/17 — Salud del dato: distribución de estados del registro (VALIDO/CORREGIDO/ANULADO) según el universo visible.
  14. 14/17 — Momentos estructurales (PAE + resultado): agrupa por tramos/etapas (momento estructural) y resume PAE + resultados por tramo.
  15. 15/17 — Acciones críticas por resultado: para acciones DURANTE, muestra cómo suelen cerrar (GANADA/PERDIDA) usando puente por fecha+dirección.
  16. 16/17 — Pensamientos críticos por resultado: para pensamientos ANTES, muestra cómo suelen cerrar (GANADA/PERDIDA) usando puente por fecha+dirección.
  17. 17/17 — Estados (E) por resultado: en DESPUÉS, clasifica estados/emociones por el resultado del mismo cierre (GANADA/PERDIDA).

Detalle completo por estadística (A qué es / B cómo se calcula / C ejemplo APB)

ESTADÍSTICA 1/17 — Cobertura por día (faltantes de los 3 registros)

A) Qué es
Mide la completitud del registro diario: para cada fecha, verifica si existe al menos un registro ANTES, uno DURANTE y uno DESPUÉS.

B) Cómo se calcula

  • Universo: registros con estado del registro = VALIDO o CORREGIDO (ANULADO excluido).
  • Unidad: DÍA (fecha).
  • Para cada fecha:
    • Se marca si hay ANTES.
    • Se marca si hay DURANTE.
    • Se marca si hay DESPUÉS.
  • Un día está “completo” si tiene los 3 momentos presentes.
  • Se reporta:
    • % de días completos.
    • lista de días con cobertura incompleta (faltantes).

C) Ejemplo APB
Día 2025-12-29:
• hay ANTES ✅
• hay DURANTE ✅
• hay DESPUÉS ✅
→ Cobertura completa (cuenta como “día completo”).

ESTADÍSTICA 2/17 — Top Pensamientos en Pérdidas (ANTES)

A) Qué es
Ranking de pensamientos (ANTES) más frecuentes en días cuyo cierre (DESPUÉS) es PERDIDA.

B) Cómo se calcula

  • Universo: “primer ANTES del día” dentro de fechas cuyo DESPUÉS (del día) = PERDIDA (según regla interna de auditoría).
  • Clasificación por resultado: el resultado se hereda desde DESPUÉS del mismo día.
  • Conteo: cada pensamiento cuenta por ocurrencia.
  • Salida: Top (ranking) + porcentajes sobre el total del universo.

C) Ejemplo APB
Si en un día con cierre PERDIDA el pensamiento ANTES fue “Tengo que entrar sí o sí”, suma 1 ocurrencia a ese pensamiento.

ESTADÍSTICA 3/17 — Top Acciones en Pérdidas (DURANTE)

A) Qué es
Ranking de acciones (DURANTE) más frecuentes en días cuyo cierre (DESPUÉS) es PERDIDA.

B) Cómo se calcula

  • Universo: “primer DURANTE del día” dentro de fechas cuyo DESPUÉS del día = PERDIDA.
  • Clasificación: resultado heredado desde DESPUÉS (mismo día).
  • Conteo: cada acción cuenta por ocurrencia (si el DURANTE trae “A, B”, suma 1 para A y 1 para B).
  • Salida: ranking + % sobre el total.

C) Ejemplo APB
Día PERDIDA → DURANTE contiene “Cumplí plan, Respeté el stop” → suma 1 a cada acción dentro de “Pérdidas”.

ESTADÍSTICA 4/17 — Top Emociones en Pérdidas (DESPUÉS)

A) Qué es
Ranking de estados/emociones (DESPUÉS) en cierres PERDIDA, más métricas de intensidad.

B) Cómo se calcula

  • Universo: registros DESPUÉS con resultado = PERDIDA.
  • Estado/Emoción: se toma del mismo DESPUÉS.
  • Ranking: conteo por ocurrencia.
  • Intensidad promedio: 
    (intensidad1 + intensidad2 + ... + intensidadN) / N
  • Picos 4–5: 
    (cantidad con intensidad ≥ 4 / N) × 100

C) Ejemplo APB
Si hay 1 DESPUÉS PERDIDA con estado “Ansiedad” e intensidad 3:
• “Ansiedad” = 1 (100%)
• Intensidad promedio = 3.0
• Picos (4–5) = 0%

ESTADÍSTICA 5/17 — Top Pensamientos en Ganadas (ANTES)

A) Qué es
Ranking de pensamientos ANTES más frecuentes en días cuyo cierre es GANADA.

B) Cómo se calcula

  • Idéntico a la 2/17, pero el universo se restringe a fechas con DESPUÉS = GANADA.
  • Resultado heredado desde DESPUÉS (mismo día).

C) Ejemplo APB
Día GANADA → pensamiento ANTES “X” → suma 1 a “X” dentro del ranking de Ganadas.

ESTADÍSTICA 6/17 — Top Acciones en Ganadas (DURANTE)

A) Qué es
Ranking de acciones DURANTE más frecuentes en días cuyo cierre es GANADA.

B) Cómo se calcula

  • Idéntico a la 3/17, pero el universo se restringe a fechas con DESPUÉS = GANADA.
  • Resultado heredado desde DESPUÉS.

C) Ejemplo APB
Día GANADA → DURANTE contiene “Cumplí plan” → suma 1 en el ranking de Ganadas.

ESTADÍSTICA 7/17 — Top Emociones en Ganadas (DESPUÉS)

A) Qué es
Ranking de estados/emociones DESPUÉS en cierres GANADA, con intensidad promedio y picos.

B) Cómo se calcula

  • Idéntico a la 4/17, pero el universo es DESPUÉS con resultado = GANADA.

C) Ejemplo APB
Si hay 1 cierre GANADA con emoción “Calma” intensidad 2:
• “Calma” = 1 (100%)
• promedio = 2.0
• picos 4–5 = 0%

ESTADÍSTICA 8/17 — Índice de Interferencia (GANADA vs PERDIDA)

A) Qué es
Compara, por resultado final del día, el % de días donde ocurrió al menos una acción DURANTE que pertenece al conjunto SET_INTERFERENCIA.

B) Cómo se calcula

  • Universo: días con al menos un DURANTE y un DESPUÉS clasificable (GANADA/PERDIDA).
  • Clasificación: resultado heredado desde DESPUÉS del día.
  • Interferencia del día: se marca “sí” si el DURANTE del día contiene alguna acción ∈ SET_INTERFERENCIA.
  • Salida: por resultado: 
    % con interferencia = (días con interferencia / días totales del resultado) × 100

C) Ejemplo APB
Si hubo 10 días GANADA y 3 de esos días tuvieron interferencia DURANTE:
• GANADA → 3/10 = 30% con interferencia.

ESTADÍSTICA 9/17 — Índice de Cumplimiento (GANADA vs PERDIDA)

A) Qué es
Igual lógica que la 8/17, pero para acciones DURANTE pertenecientes al conjunto SET_CUMPLIMIENTO.

B) Cómo se calcula

  • Universo: días con DURANTE y DESPUÉS clasificable.
  • Cumplimiento del día: “sí” si DURANTE contiene alguna acción ∈ SET_CUMPLIMIENTO.
  • Salida: por resultado: 
    % con cumplimiento = (días con cumplimiento / días totales del resultado) × 100

C) Ejemplo APB
10 días GANADA; 7 con cumplimiento → 70% cumplimiento en GANADAS.

ESTADÍSTICA 10/17 — Balanza Cumplimiento vs Interferencia

A) Qué es
Presenta, para cada resultado (GANADA/PERDIDA), ambos porcentajes (cumplimiento e interferencia) en una misma vista para comparación visual rápida.

B) Cómo se calcula

  • Reutiliza los cálculos de 8/17 y 9/17 (misma definición de día, mismo universo).
  • Salida: tabla “Resultado → % Cumplimiento / % Interferencia”.

C) Ejemplo APB
GANADA: Cumplimiento 70% / Interferencia 20% → permite ver “balanza” sin interpretación causal.

ESTADÍSTICA 11/17 — Top Interferencias por Resultado (DURANTE)

A) Qué es
Lista qué acciones interferentes (DURANTE, acciones ∈ SET_INTERFERENCIA) aparecen cuando el resultado es GANADA y cuando es PERDIDA.

B) Cómo se calcula

  • Universo: registros DURANTE asociados a días con DESPUÉS clasificable (GANADA/PERDIDA).
  • Filtro: se toman solo acciones ∈ SET_INTERFERENCIA.
  • Clasificación: resultado heredado del DESPUÉS del día.
  • Salida: conteo descriptivo (no necesariamente “ranking duro” si el universo es chico).

C) Ejemplo APB
PERDIDAS: “Moví stop” aparece 2 veces; GANADAS: no aparece → se muestra como desglose, sin causalidad.

ESTADÍSTICA 12/17 — Intensidad promedio + picos por resultado

A) Qué es
Compara la intensidad emocional DESPUÉS entre cierres GANADA y PERDIDA.

B) Cómo se calcula

  • Universo: registros DESPUÉS con resultado GANADA o PERDIDA y con intensidad válida (típicamente 1–5).
  • Para cada resultado:
    • Promedio de intensidad.
    • % picos (intensidad ≥ 4).

C) Ejemplo APB
GANADA: intensidades [1,2] → promedio 1.5; picos 0%.
PERDIDA: intensidades [4,5] → promedio 4.5; picos 100%.

ESTADÍSTICA 13/17 — Salud del dato

A) Qué es
Auditoría del dataset: muestra la distribución de registros según su estado (VALIDO/CORREGIDO/ANULADO) en el universo visible por filtros.

B) Cómo se calcula

  • Universo: registros visibles según filtros activos (sin reinterpretar el contenido).
  • Conteo: cuántos están en cada estado.
  • Porcentaje: (cantidad de estado / total) × 100.

C) Ejemplo APB
Si hay 8 registros visibles: 3 VALIDO, 4 CORREGIDO, 1 ANULADO →
• VALIDO 37.5% • CORREGIDO 50% • ANULADO 12.5%.

ESTADÍSTICA 14/17 — Momentos estructurales (PAE + resultado)

A) Qué es
Agrupa el historial por momentos estructurales (tramos/etapas consecutivas) y, dentro de cada tramo, resume:

  • Resultados operativos (DESPUÉS) del tramo.
  • Cadena PAE por resultado (pensamientos ANTES, acciones DURANTE, estados DESPUÉS).
  • Intensidad promedio y picos en DESPUÉS del tramo.

B) Cómo se calcula

  • Universo: registros válidos (ANULADO excluido). Si filtros dejan vacío, se usa el universo completo (según implementación específica de esta estadística).
  • Segmentación: el historial se parte en tramos según la etiqueta “momento estructural”.
  • Capa 2 (resultado): se calcula usando SOLO DESPUÉS del tramo.
  • Capa 3 (cadena PAE): ANTES y DURANTE se asocian a fechas que tienen DESPUÉS dentro del mismo tramo (consistencia interna).
  • Top fijo: se reservan posiciones aunque falten datos (integridad estructural).

C) Ejemplo APB
Tramo “AJUSTE_ESTRATEGIA” con 2 cierres DESPUÉS: 1 GANADA y 1 PERDIDA →
• Resultado del tramo: 50%/50%
• Cadena PAE: pensamientos/acciones/estados se listan por resultado dentro del tramo.

ESTADÍSTICA 15/17 — Acciones críticas por resultado

A) Qué es
Toma acciones DURANTE y responde: “Cuando aparece esta acción, ¿cómo suele cerrar la operación?” clasificada por GANADA/PERDIDA.
Esta estadística usa un puente por (fecha + dirección) para evitar mezclar COMPRA/VENTA del mismo día.

B) Cómo se calcula

  • Universo de clasificación (puente):
    • Para cada par (FECHA + DIRECCIÓN), se busca el último registro DESPUÉS del día con resultado GANADA/PERDIDA.
    • Ese resultado se guarda como “cierre” del par (día+dirección).
    • Si un par no tiene DESPUÉS GANADA/PERDIDA, no entra al universo (no se puede clasificar).
  • Datos analizados: acciones extraídas desde registros DURANTE (respetan filtros) que pertenecen a pares (día+dirección) con cierre válido.
  • Conteo: por ocurrencias de acción.
  • Salida: Top 5 acciones + fila de totales; por cada acción:
    • GANADAS: cantidad y % sobre el total de esa acción.
    • PERDIDAS: cantidad y % sobre el total de esa acción.
    • TOTAL: ocurrencias (100% por fila).

C) Ejemplo APB
Día 2025-12-29, dirección COMPRA:
• último DESPUÉS = GANADA
• DURANTE contiene “Cumplí plan”
→ “Cumplí plan” suma 1 en GANADAS.

ESTADÍSTICA 16/17 — Pensamientos críticos por resultado

A) Qué es
Toma pensamientos ANTES y responde: “Cuando aparece este pensamiento, ¿cómo suele cerrar la operación?” clasificada por GANADA/PERDIDA, usando puente por fecha + dirección.

B) Cómo se calcula

  • Puente: idéntico al de la 15/17:
    • Para cada (fecha+dirección), se usa el último DESPUÉS con GANADA/PERDIDA.
  • Datos analizados: pensamientos extraídos desde ANTES (respetan filtros) y solo si su (fecha+dirección) tiene puente válido.
  • Conteo: por ocurrencias de pensamiento.
  • Salida: Top 5 + totales (mismo formato APB que la 15/17).

C) Ejemplo APB
Día 2025-12-29, dirección VENTA:
• último DESPUÉS = PERDIDA
• ANTES contiene “Tengo que entrar sí o sí”
→ ese pensamiento suma 1 en PERDIDAS.

ESTADÍSTICA 17/17 — Estados (E) por resultado

A) Qué es
Toma estados/emociones DESPUÉS y responde: “Cuando aparece este Estado (E) en DESPUÉS, ¿cómo cerró la operación?” usando el resultado del mismo registro DESPUÉS (sin puente).

B) Cómo se calcula

  • Universo: registros DESPUÉS válidos (respeta filtros activos).
  • Clasificables: solo DESPUÉS cuyo resultado sea GANADA o PERDIDA.
  • Estado (E): se extrae del mismo registro (campo estado/emoción; puede ser lista).
  • Conteo: por ocurrencias de estado.
  • Salida: Top 5 + totales; por estado:
    • GANADAS: cantidad + % sobre el total del estado.
    • PERDIDAS: cantidad + % sobre el total del estado.

C) Ejemplo APB
Si “Ansiedad” aparece en 2 cierres DESPUÉS:
• 1 cierre GANADA, 1 cierre PERDIDA
→ “Ansiedad”: GANADAS 1 (50%), PERDIDAS 1 (50%), TOTAL 2 (100%).

Errores comunes de interpretación (por favor leer)

  • 100% con 1 dato no significa patrón estable.
  • Más frecuencia no significa “mejor” ni “peor”. Solo significa “más repetido”.
  • Co-ocurrencia no implica que algo “cause” el resultado.
  • Una estadística aislada no se interpreta sola: primero revisá universo, filtros y tamaño de muestra.

Nota final

Este documento está diseñado para que el usuario pueda auditar cada número: universo → filtro → reglas → fórmula → salida.
Si una etiqueta, campo o filtro no existe en un registro, ese registro puede quedar fuera del universo de una estadística específica (por ejemplo, falta dirección en estadísticas que puentean por fecha+dirección).
